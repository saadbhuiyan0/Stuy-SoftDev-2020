What is Less?

Less, or Leaner Style Sheets, is a dynamic preprocessor style sheet language.
A preprocessor is a program that processes input data and produces output that is input to another program.
In this case, the input is Less code which then produces CSS code.


But why should you care? Because CSS sucks.

CSS is so repetitive. You constantly declare the same style rules and it get annoying and frustrating.
It's even worse when you have to go back and change that style because you have to change it everywhere you declared it!
Good luck trying to read hex codes.
As you add more and more styles to your CSS, it gets more and more confusing to traverse through all the layers.
This endless cycle of copying and pasting is terrible for scalability and maintainability.


So where does Less fit into all this?

That's a great question (I just asked on your behalf)!
See, Less is a CSS preprocessor. Exciting!
Mozilla Developer Network Web Docs defines a CSS preprocessor as “a program that lets you generate CSS from the preprocessor's own unique syntax.”
Essentially you would write input code, in this case Less code, and then compile it into CSS.

Less simply allows you to write better code.
It extends the default capabilities of CSS so you can:
  • Have an easier time implementing your CSS across different platforms.
  • Organize/modularize your code and thus repeat yourself less.
  • Scale up, update, and maintain the codebase with far less hassle.

Variables and Mathematical Operations

You're a CS student, you know what a variable is.
In Less, variables allow you to store all your commonly used values in one location lending to modularity.
Assigning values that you use repeatedly to variables has many advantages:
variables are far easier to read and remember than numbers and hexcodes or whatever so it is easier for everyone in your group when your project's stylesheet grows larger;
if you ever want to change the value everywhere, you only need to change it one place;
and who doesn't love variables, they just make life easy.
In this scenario, I have variables for my primary color and primary background color.
If I ever need to update the color scheme for the website, I just need to go to one place instead of 50! 

                        /* Input Less Code */

                            @primary-color: #FF00FF;
                            @primary-bg: #00FF00;

                            body {
                                color: @primary-color;
                                background: @primary-bg;
                            }

                            #header {
                                color: @primary-color;
                            }
                        

                        /* Output CSS Code */

                            body {
                                color: #FF00FF;
                                background: #00FF00;
                            }

                            #header {
                                color: #FF00FF;
                            }
Variables are a great way to manipulate numbers that are relative to each other.
If you want something to always be twice as large as something, or a certain amount larger, or smaller, or whatever it may be, you can use mathematical operations.
You can play around with the values of variables, numbers, and colors with all basic math operations and more you can find in the documentation for functions. In this case, my width variable is dependent on the height variable. So the canvas is always twice as wide as it is high and all I need to change is the height and the ratio is unaffected.
Less independent declarations, less repetitiveness, easy to work with. 

                        /* Input Less Code */ 

                            @primary-color: #900C3F;
                            @height: 100;
                            @width: @height * 2;

                            canvas {
                                color: @primary-color;
                                height: @height;
                                width: @width;
                            }
                        

                        /* Output CSS Code */ 

                            canvas {
                                color: #900C3F;
                                height: 100;
                                width: 200;
                            }
Another way to use variables is to create shortcuts/placeholders for paths, URLS, or whatever else you can come up with.
Having a variable for the images folder allows me to reference it everytime I want an image and not have to write the whole path to the folder.
This also means I only need to the change the path in one place if I move the folder and not in every declaration. 

                        /* Input Less Code */ 

                            @images: “../app/bin/img/”

                            header {
                                background-image: url(“@{images}/nicehead.jpg);
                            }

                            body {
                                background-image: url(“@{images}/goodbody.jpg);
                            }
                        

                        /* Output CSS Code */ 

                            header {
                                background-image: url(“../app/bin/img/nicehead.jpg);
                            }

                            body {
                                background-image: url(“../app/bin/img/goodbody.jpg);
                            }


Nesting

Nesting allows you to make less independent declarations and create concise rulesets.
You can nest the rules for different elements so the element inherits styles according to the hierarchy.
It is pretty simple to figure out how it works with a little messing around.
This code sets style rules for a multiple elements, but since my general style and theme is the same I can have inheritance do a lot of the work for me.
All the elements that inherit styles from a parent will all be updated if I change a rule in the parent.
Using nesting allows for much less repetitive code than usual as you make far less declarations. 

                        /* Input Less Code */ 

                            #container {
                                border:1px solid #ddd;
                                &.dark-border { border:1px solid #333; }
                                h1 { 
                                    color:#333; 
                                    text-align:center; 
                                }
                                p { color:@blue;}
                            }
                        

                        /* Output CSS Code */ 

                            #container {
                                border: 1px solid #ddd;
                            }

                            #container.dark-border {
                                border: 1px solid #333;
                            }

                            #container h1 {
                                color: #333;
                                text-align: center;
                            }

                            #container p {
                                color: #0098cb;
                            }


Mixins

Mixins allow you combine rulesets from different classes and ids.
If you include another class/id in the styles for one class/id, it will inherit all the styles from the parent you're mixing in.
That means less lines of codes and less repetitive declarations.
If I want all my children rectangles to have the same color and border, I will have every rectangle class inherit the original parent rectangle's style.
This way if I want to change how a rectangle behaves, I can change it in the parent class and the changes will be reflected across all rectangles.
That makes for simple, modular, and very easy to maintain code. 

                        /* Input Less Code */ 

                            #rectangle {
                                background-color: #4CAF50;
                                border-radius: 100%;
                            }
                            
                            #small-rectangle {
                                width: 50px;
                                height: 50px;
                                #rectangle
                            }

                            #big-rectangle {
                                width: 100px;
                                height: 100px;
                                #rectangle
                            }
                        

                        /* Output CSS Code */ 

                            #rectangle {
                                background-color: #4CAF50;
                                border-radius: 100%;
                            }
                            
                            #small-rectangle {
                                width: 50px;
                                height: 50px;
                                background-color: #4CAF50;
                                border-radius: 100%;
                            }

                            #big-rectangle {
                                width: 100px;
                                height: 100px;
                                background-color: #4CAF50;
                                border-radius: 100%;
                            }
You can also use inputs when mixing your rulesets.
In this case, size is a variable which defaults to 50px.
But when making a big rectangle using the same rules as the rectangle, I can input a new size and Less will change the values for me.
This is super convenient and simple to work with. 

                        /* Input Less Code */

                            #rectangle(@size: 50px) {
                                background-color: #4CAF50;
                                border-radius: 100%;
                                width: @size;
                                height: @size;
                            }
                            
                            #big-rectangle {
                                #rectangle(100px)  
                            }
                        

                        /* Output CSS Code */

                            #rectangle {
                                background-color: #4CAF50;
                                border-radius: 100%;
                                width: 50px;
                                height: 50px;
                            }

                            #big-rectangle {
                                background-color: #4CAF50;
                                border-radius: 100%;
                                width: 100px;
                                height: 100px;
                            }


Logic

Less also allows you to use logic to generate style rules.
You can mess around with if, when, and conditionals to make your life easier.
You can use loops to automatically generate different divs with respective styles so you don’t have to write them all individually.
You can use if statements to apply certain styles only under certain conditions.
For example, instead of me individually deciding if the background is light or dark and changing the text color appropriately, I will let Less figure it out.
I use the built-in Less function lightness(color) and made two conditions, if the background color is light then the text color is black, and if the background color is dark, the text color is white. 

                        /* Input Less Code */

                            .text-color (@bg-color) when (lightness(@bg-color) >= 50%) { 
                                color: black;
                            }

                            .text-color (@bg-color) when (lightness(@bg-color) < 50%) { 
                                color: white;
                            }

                            .text-color (@bg-color) { background-color: @bg-color; }

                            .div-1 { .text-color (yellow); }
                            .div-2 { .text-color (brown; }
                        

                        /* Output CSS Code */

                            .div-1 { 
                                background-color: yellow;
                                color: black; 
                            }
                            .div-2 { 
                                background-color: brown;
                                color: white; 
                            }
There are many other built-in functions you can mess around with including these color functions:

                lighten(@color, @amount)
                darken(@color, @amount)
                fadein(@color, @amount)
                fadeout(@color, @amount)
                fade(@color, @amount)
                spin(@color, @amount)
